<product_requirements>
The user requires a web application to upload CSV dataframes containing de-identified health records (specifically discharge summaries). The application needs to extract sections, split them into individual sentences, and allow multiple users (annotators) to add tags based on the five domains of Social Determinants of Health (SDOH). Key features include:
1.  **User Authentication:** A full user authentication system.
2.  **Annotation Collection:** Collect individual annotations for each sentence.
3.  **Metrics/Analytics:** A page to display inter-rater reliability, tag prevalence, and other relevant metrics (documents, sentences, annotations, tagged, skipped, annotators).
4.  **Structured Tags:** Ability to parse a Word document containing domain headings and bullet point subheadings to create a multi-select tag interface.
5.  **Valence Selection:** For each tag, an option to input positive or negative valence (e.g., presence or absence of a relationship).
6.  **Skip Functionality:** A Skip or No SDOH Content button to move to the next sentence if none of the domains apply, and automatically advance to the next sentence, returning to the top of the page.
7.  **Admin Functionality:**
    *   Admin accounts to upload CSV files for all annotators.
    *   Admin user management (create, view, activate/deactivate, delete users, assign roles).
    *   Centralized document management for admins (upload, delete).
    *   Admin should be able to download the annotated CSV file.
8.  **UI/UX Enhancements:**
    *   Horizontal toolbar for navigation tabs (Manage, Upload, Documents, Annotate, Analytics).
    *   Confirmation pop-up for user/document deletion.
9.  **Annotation Persistence:** Annotations should be tied to individual sentences and reset when navigating between sentences. After annotating the last sentence, it should return to the home page (Documents tab).
10. **Consistent Annotator ID:** All annotations should be associated with the currently logged-in user.
</product_requirements>
